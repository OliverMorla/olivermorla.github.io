HTML Home

- HTML Stands for Hyper Text Markup Language
- A Markup Language is a language that can formatt or structure a document.
- HTML is the standard markup language for structuring and formatting webpages.


HTML Introduction

- HTML consists of a series of elements.
- HTML elements tell the browser how to structure the webpage.

Example


- The !DOCTYPE html declaration defines what type of document you are working on.
- The html element is the root/start element of an html page.
- The head element is where all meta information is stored for the html page.
- The title element defines the title of the webapge (which is shown on the tab)
- The body element defines what is shown on the webpage.
- The h1 element defines large and important heading.
- The p element defines a paragraph.

- The html element is defined with a start tag <TagName> and a ending that </TagName>
- Some HTML elements have no content (like the br element). These elements are called empty elements. Empty elements do not have an end tag!
- The purpose of Web Browsers is to read an html document and display it correctly.


HTML Editors

- For HTML all you need is a simple text-editor.
- Make sure to format file as .html or .htm and set encoding to UTF-8
- UTF-8 Encoding is preferred for HTML.

HTML Links

- HTML Links/HyperLinks are defined using the <a> or in other words anchor.
- It requres a href attribute. This attribute specifies the URL of the page the link goes to.
- For example:
<a href=""> This is is a link </a>

HTML Images

- HTML images are defined with the <img> element or in other words image.
- It requires a src and alt attribute. The src attribute specifies the path to the image to be displayed.
- width width and height Attributes are optional.
- For example:
<img src="" alt="">
- Absolute URL (External) & Relative URL (Internal)
- Tip: It is almost always best to use relative URLs. They will not break if you change domain.
- The required alt attribute for the tag specifies an alternate text for an image, if the image for some reason cannot be displayed. This can be due to a slow connection, or an error in the src attribute, or if the user uses a screen reader.

Empty HTML Elements

- HTML elements with no content are considered empty elements
- The HTML standard does not require lowercase tags, but W3C recommends lowercase in HTML, and demands lowercase for stricter document types like XHTML.


HTML Attributes

- HTML attributes provide additional infomration from the elements, or allow us to manipulate the element.
- All HTML elements can have attributes.
- Attributes always specified in the start tag
- Attributes usually come in name/value pairs like: name="value"
- The style attribute is used to add styles to an element, such as color, font, size, and more. - You should always include the lang attribute inside the tag, to declare the language of the Web page. This is meant to assist search engines and browsers. - Country codes can also be added to the language code in the lang attribute. So, the first two characters define the language of the HTML page, and the last two characters define the country. - The title attribute defines some extra information about an element. The value of the title attribute will be displayed as a tooltip when you mouse over the element: - However, W3C recommends quotes in HTML, and demands quotes for stricter document types like XHTML. - Double quotes around attribute values are the most common in HTML, but single quotes can also be used.

Summary

All HTML elements can have attributes
The href attribute of < a > specifies the URL of the page the link goes to
The src attribute of < img > specifies the path to the image to be displayed
The width and height attributes of < img > provide size information for images
The alt attribute of < img > provides an alternate text for an image
The style attribute is used to add styles to an element, such as color, font, size, and more
The lang attribute of the < html > tag declares the language of the Web page
The title attribute defines some extra information about an element


HTML Headings

- HTML headings are titles or subtitles.
- Browsers automatically add some white space (a margin) before and after a heading.
- Search engines use the headings to index the structure and content of your web pages.
- Use HTML headings for headings only. Don't use headings to make text BIG or bold.


HTML Paragraphs

- A paragraph always starts on a new line, and browsers automatically add some white space (a margin) before and after a paragraph.
- Large or small screens, and resized windows will create different results.

HTML Horizontal Rules

- The < hr > tag defines a thematic break in an HTML page used to separate content, and is most often displayed as a horizontal rule.

HTML Line Breaks

- The HTML < br > element defines a line break (new line) without starting new paragraph.

HTML pre element

- The HTML < pre > element defines preformatted text.
- The text inside a < pre > element is displayed in a fixed-width font (usually Courier), and it preserves both spaces and line breaks:
